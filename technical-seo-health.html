Step-by-step technical SEO analysis framework

This workflow is designed for one exact enterprise ecommerce category. Repeat it by category, then compare categories to find systemic platform problems.

1

Define category universe

Collect all URLs belonging to the category: main landing page, paginated URLs, filter URLs, brand/category combinations, product listing pages, and historical URLs from GSC/logs.

2

Classify URL types

Tag each URL as indexable category, strategic facet, UX-only facet, pagination, product, redirect, error, duplicate, parameter, or orphan.

3

Crawl like Googlebot

Run JS and non-JS crawls, mobile user-agent crawl, sitemap crawl, list crawl from GSC exports, and compare discoverability gaps.

4

Join data sources

Merge crawl data, GSC performance, index coverage, logs, GA4 revenue/conversions, CWV and internal BI data by normalized canonical URL.

5

Measure health metrics

Calculate indexability, crawl efficiency, canonical consistency, sitemap hygiene, CWV pass rate, structured data validity and internal link depth.

6

Find root causes

Separate issue symptoms from causes: template logic, parameter generation, JavaScript rendering, broken canonicals, poor linking, duplicate content, server errors.

7

Prioritize by business risk

Score issues by affected organic landing pages, demand, revenue importance, crawl/index impact and implementation effort.

8

Track after release

Monitor before/after movement in crawl behavior, indexed URLs, impressions, clicks, rankings, CWV and conversion metrics for at least 28–90 days.

Recommended workflow rhythm

Enterprise SEO is not one audit PDF per year. It is a monitoring system with recurring diagnostics.

Daily / automated

5xx spikes, robots.txt changes, sitemap failures, noindex increases, canonical anomalies, crawl drops, template deployment alerts.

Weekly

Category score trend, GSC clicks/impressions, crawl waste, top issue queue, CWV group changes, internal linking anomalies.

Monthly / quarterly

Full category crawl, log-file analysis, facet indexation review, competitor technical comparison, platform-level technical debt roadmap.

Recommended data stack

For an enterprise ecommerce category, Looker Studio can visualize the dashboard, but BigQuery or another warehouse should hold the joined URL-level data model.

1. Search data

  • Google Search Console API
  • URL Inspection API samples
  • GSC CWV report
  • Rank tracking / Semrush / Ahrefs

2. Crawl data

  • Screaming Frog SEO Spider
  • Sitebulb / Botify / Oncrawl
  • XML sitemap exports
  • Rendered HTML extraction

3. Log data

  • CDN logs
  • Server logs
  • Googlebot verification
  • Log analyzer pipeline

4. Performance data

  • CrUX
  • PageSpeed Insights API
  • Real-user monitoring
  • Lighthouse for lab debugging

5. Business data

  • GA4 / ecommerce events
  • Internal BI revenue
  • Margin / category priority
  • Product availability

Definition of done for technical SEO fixes

A fix is not done when Jira is closed. A fix is done when Google’s behavior and organic KPIs confirm it.

Engineering done

Code released, QA passed, production crawl confirms expected technical state.

Google discovery done

Googlebot crawls updated URLs and no longer spends material crawl volume on the old problem.

Indexation done

Index coverage and canonical selection reflect the intended URL policy.

Business done

Relevant impressions, clicks, rankings, conversions or revenue trend in the expected direction.
